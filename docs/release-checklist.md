# Release Checklist

Use this before pushing the first public version.

## Identity And Privacy

- repository owner name is the intended public identity
- no old private username remains in docs or copied files
- no personal email, phone, or unrelated account identifiers remain
- avatar and public profile are acceptable for public release

## Scope

- the repository only contains the intended first public batch
- withheld internal-only skills are not mixed in accidentally
- Unity showcase remains framed as a domain pack, not the whole project

## Repository Basics

- `README.md` explains the project in the first screen
- `SKILL_TREE.md` matches the shipped skill set
- `SKILL_TEMPLATE.md` is present
- contribution guidance exists
- examples exist

## Skill Quality

- every shipped skill has the required metadata fields
- copied references still resolve
- no skill claims authority it should not own
- example chains match the skills that are actually included

## Public Packaging

- repository name matches the intended positioning
- short description is ready
- topics are chosen
- first commit message is clear
- license choice has been made explicitly

## Final Sanity Check

- read the repo once as a new user
- ask whether the first five minutes are clear
- if not, simplify before publishing

This is the first public slice of the governed skill tree project.

It is not a full internal dump.
It is a deliberately scoped release intended to show the core structure, governance logic, and one domain-oriented showcase.

## What This Release Is Trying To Show

This release is meant to show that a skill system becomes easier to maintain and extend when the following responsibilities are treated as separate layers:

- capability routing
- phase control
- tool routing
- domain execution
- verification
- compression / handoff / storage

The main claim is not that more skills are better.

The main claim is that explicit governance becomes more valuable as a skill tree grows and collaboration becomes longer-horizon.

---

## What This Release Does Not Include

This release does **not** include:

- the full internal skill inventory
- every private or domain-specific skill
- a benchmark suite
- a complete plugin or agent framework
- a claim that the system is finished

This is a public first slice, not a final system dump.

---

## Why This Version Matters

Without governance, skill collections tend to drift toward:

- flat accumulation
- fuzzy triggers
- weak phase discipline
- unclear authority boundaries
- reactive growth
- brittle continuity across sessions

This release introduces a structure intended to reduce those problems.

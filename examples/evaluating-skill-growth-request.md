# Example: Skill Growth Proposal

## User Request

"We keep solving the same kind of problem by hand. Help me decide whether this should become a real skill."

## Recommended Chain

1. `brain-skill-orchestrator`
2. `controller-work-state-controller`
3. `brain-capability-gap-detect`
4. `balance-assumption-audit`
5. `SKILL_TEMPLATE.md` authoring workflow after approval

## Why

- the problem starts as a governance question, not a writing task
- the system should distinguish real capability gaps from naming noise
- approval should happen before a new public skill is authored
- the template should be the final authoring surface, not the first diagnostic step

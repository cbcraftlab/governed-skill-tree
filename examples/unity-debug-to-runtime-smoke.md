# Example: Unity Debug To Runtime Smoke

## User Request

"The Unity path still fails. Diagnose it, then verify it can actually run."

## Recommended Chain

1. `brain-skill-orchestrator`
2. `controller-work-state-controller`
3. `balance-unity-codex-guard`
4. `eye-unity-log-debug` or `eye-unity-scene-audit`
5. `leg-unity-runtime-smoke`

## Why

- risky Unity edits should not start without guardrails
- the main uncertainty is usually evidence first, not code first
- compile success is not the same as runtime success

# Bootstrap Checklist

Use this reference for a fast orientation pass in a new repo or worktree.

## First-Pass Checklist

- top-level directories
- manifest files or project markers
- main runtime or framework clues
- test locations
- build or launch hints
- local tooling or skill folders

## Common Project Clues

### Unity

Look for:

- `Assets/`
- `ProjectSettings/`
- scene folders
- package manifests
- editor automation or MCP support

### Web or Node

Look for:

- `package.json`
- framework config files
- app entry folders
- test runners

### Python

Look for:

- `pyproject.toml`
- `requirements.txt`
- package directories
- scripts or CLI entry points

## Stop Rule

Bootstrap ends when you can answer:

- what kind of project this is
- where changes likely belong
- how it is probably run or validated
- what the next inspection step should be

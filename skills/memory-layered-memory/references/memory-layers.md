# Memory Layers

Use this reference when the correct memory layer is unclear.

## Short-Term

Store here:

- current task status
- latest smoke or validation results
- current blockers
- immediate next step

Do not store here:

- stable architecture rules
- durable user preferences

## Long-Term

Store here:

- collaboration habits
- formatting or delivery preferences
- stable lessons likely to matter again

Do not store here:

- one debugging round's temporary state
- project-specific baselines that belong to project knowledge

## Project Knowledge

Store here:

- stable system baselines
- durable module boundaries
- fixed constraints
- known entry chains or protocol truths

Do not store here:

- current rollout progress
- personal working preferences

## Mixed Memories

If one note mixes layers, split it instead of forcing it into one bucket.

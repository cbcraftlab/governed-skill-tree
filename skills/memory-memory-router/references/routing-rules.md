# Memory Routing Rules

Use this reference when a memory-related request could match more than one memory skill.

## Quick Distinctions

### Layered Memory

Choose when the user wants:

- save
- recall
- update
- delete
- choose the correct layer

### Codex Handoff

Choose when the user wants:

- next-window continuation
- account-switch continuity
- copy-ready resume note

### Status Compressor

Choose when the user wants:

- the short version
- current status
- a compact update right now

### Decision Journal

Choose when the user wants:

- why a decision was made
- which option was rejected
- what constraint forced the choice

## Non-Memory Cases

Do not route to memory when:

- the user is just continuing the same conversation
- the request is a normal implementation step
- nothing needs to be stored, recalled, or compressed
